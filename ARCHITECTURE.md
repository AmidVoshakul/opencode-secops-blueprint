# Architecture

High-level system design of the OpenCode SecOps Blueprint — a drop-in configuration framework for the OpenCode AI coding assistant.

---

## 🔄 Data Flow

```
User launches OpenCode
    ↓
opencode.json loads MCP servers + permissions
    ↓
instructions/*.md loaded into session context
    ↓
User runs /init → AGENTS.md generated (project-specific)
    ↓
User works with commands (/architect, /pro-test, /guard, etc.)
    ↓
Commands invoke specialized agents (test-engineer, security-auditor, etc.)
    ↓
Agents operate in isolated subtask contexts
    ↓
Results returned to main session via Context Bridge
```

## 🧩 Component Boundaries

### Configuration Layer (`opencode.json`)
- MCP server definitions (12 servers)
- Permission matrix (Git operations = `"ask"`, skills = allowlist + `"*": "deny"`)
- Instruction file paths

### Command Layer (`commands/`)
- 13 Markdown-based slash commands
- Each command specifies: `description`, `agent`, `subtask`
- Commands are prompts consumed by OpenCode, not executable scripts

### Agent Layer (`agents/`)
- 4 specialized sub-agents with isolated tool access
- Each agent defines: `mode`, `tools` (with wildcards), system prompt
- Agents run in separate context windows — no pollution of main session

### Rules Layer (`instructions/`)
- 5 instruction files loaded into every session
- Cover: engineering standards, security, language-specific rules, testing, documentation
- Enforced by all agents and commands

### Security Layer (`hooks/`)
- Pre-commit secret detection hook
- Scans for: GitHub PAT, OpenAI keys, AWS keys, JWT, private keys, hardcoded passwords

## 🔐 Security Model

### Permission Matrix
| Operation | Default | Rationale |
|---|---|---|
| `git_git_commit` | `ask` | Prevent accidental commits |
| `git_git_push` | `ask` | Prevent accidental pushes |
| `git_git_add` | `ask` | Prevent staging unwanted files |
| `git_git_merge` | `ask` | Prevent accidental merges |
| `git_git_pull` | `ask` | Prevent overwriting local changes |
| `git_git_create_branch` | `ask` | Prevent branch pollution |
| `git_git_checkout` | `ask` | Prevent unwanted branch switches |
| `git_git_reset` | `ask` | Prevent losing staged changes |
| `skill: *` | `deny` | Block unverified community skills |

### Agent Tool Isolation
| Agent | Write | Edit | Bash | Rationale |
|---|---|---|---|---|
| `security-auditor` | ❌ | ❌ | ✅ | Can run audits, cannot modify code |
| `docs-writer` | ✅ | ✅ | ❌ | Can write docs, cannot run commands |
| `test-engineer` | ✅ | ✅ | ✅ | Full access for test lifecycle |
| `code-reviewer` | ❌ | ❌ | ✅ | Can analyze, cannot modify |

## 🌐 MCP Server Architecture

All 12 MCP servers are configured as `local` type — they run as child processes of OpenCode.

| Server | Transport | Purpose |
|---|---|---|
| `time` | `uvx` | Temporal context (2026 anchor) |
| `searxng` | `npx` | Private web search (self-hosted) |
| `sequential-thinking` | `uvx` | Complex reasoning chains |
| `filesystem` | `npx` | File operations within workspace |
| `fetch` | `uvx` | URL content retrieval |
| `chrome-devtools` | `npx` | Browser automation and debugging |
| `playwright_mcp` | `npx` | E2E testing automation |
| `context7` | `npx` | Library documentation lookup |
| `memory` | `npx` | Session knowledge graph |
| `git` | `uvx` | Version control via MCP (not bash) |
| `code-index` | `uvx` | Deep symbol indexing and search |
| `github` | `npx` | PR, issues, repo operations |
